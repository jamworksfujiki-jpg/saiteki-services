2-2. 実現できないこと（Web/外部プロジェクトでの制約）

項目制約代替策
SF Pro等のAppleシステムフォント Webサイトへの直接埋め込み・再配布はライセンス上不可 CSS で -apple-system, BlinkMacSystemFont, "Hiragino Sans", "Yu Gothic UI", sans-serif のスタックを使う
SF Symbols（6,900+ アイコン） Web/HTMLでの利用は原則不可（Apple製アプリ画面内のみ） Phosphor Icons / Lucide / Heroicons / Material Symbols で代替
Dynamic Color（ユーザー壁紙からの自動生成） Android/iOS のシステム機能。Webでは再現不可 固定 color roles を CSS 変数化する。ダーク/ライト切替のみ prefers-color-scheme で対応
Dynamic Type（11pt〜AX5の自動拡大） Webでは完全再現不可 rem 単位 + メディアクエリで拡大対応、ユーザーのブラウザ拡大を妨げない設計
Materials（ultraThin/thin/regular/thick/chrome） Web では backdrop-filter: blur() で類似実装可能だが Safari/iOS で挙動差あり 背景に半透明色 + backdrop-filter: blur(20px) saturate(180%) で近似
Continuous corner（squircle） CSS の border-radius は厳密には circular。iOS は continuous SVG でsquircleパスを使うか、CSS Houdini Paint API 使用。一般プロジェクトでは border-radius で十分近似
Liquid Glass（Apple Icon Composer / Sequoia+） macOS/iOS 専用機能、Webで再現不可 イメージとしてグラデ + ハイライトで模倣（厳密な再現はあきらめる）
Live Activities, App Clips 等のTechnology Templates プラットフォーム機能。デザイン素材は配布されているが、機能としてはApple純正アプリのみ LP 上はスクリーンショットや Product Bezels を使ったモックで表現

4. 推奨アクション

定期的に更新: HIGとM3は年1回程度大型改訂される（M3 Expressive 2025、SF Symbols 7、iOS 26など）。3-6ヶ月ごとにこのレポートと CLAUDE.md を再生成し、最新値を反映する。
SPA対応の取得方法: 今後 HIG/M3 から最新情報を取得する際は、WebFetch ではなく Playwright で実際にレンダリングしてテキストを抜き出す方法に切り替える（本レポート作成後に実装可）。
個別案件への適用: ブランドカラー・トーンが指定されている案件（サイテキ等）では、A/B章の構造（タイポスケール・gridシステム・state layer・モーション duration）はそのまま採用し、配色のみブランド準拠に置き換える運用が安全。

5. 結論

Apple HIG / Material Design 3 / Apple Design Resources の中核となる実装値はすべて CLAUDE.md（A〜D章）に保存済み。SPA制限により公式ドキュメントの本文ニュアンスは一部欠落しているが、数値・トークン・コンポーネント仕様はGoogle/Apple の一次資料から補完しており、実装に必要な情報は揃っている。

今後すべてのUI/LP作成時には、新規ファイルを書き始める前に必ず CLAUDE.md の「デザイン方針（全プロジェクト共通）」章を読み返し、コンポーネント実装時は仕様の数値を逐一引用する運用を徹底する。
